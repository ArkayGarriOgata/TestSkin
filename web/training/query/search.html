The Query Editor

aMs presents the Query Editor dialog when you select Search from the Shortcuts menu, click the Find (magnifier icon) on a record listing, or when the Query Editor radio button is chosen from the ubiquitous "Find records in the Xxxxx table where" dialog.

Complete documentation on the Query Editor can be found here.

Basic Query

Demonstrates using the Query Editor to find records in a table that share a common value in a field. In this example will find inventory that shares the same customer id.

Updated 02/12/19

Query on Related Fields

Demonstrates using fields from a related table to filter records. This example will find inventory by customer name that was glued before a given date. Note that for this to work there needs to be an explicit relationship between the table of interest and the table of the external field. Diagrams showing these relations can be found on the aMs Relations page.

Saving and Loading Queries

Demonstrates using the Query Editor Save and Load commands that are available under the Action (little gear) icon. Note that the queries you save are specific to the table that you selected as the master table; therefore, the name given to the saved query should in someway indicate the master table, for instance a prefix.

Updated 02/12/19

Query Selection Options

Demonstrates applying addition criteria to the current selection of records as well as excluding those that meet a condition. When the number of records in the current selection is small, you may be able to leverage sorting and command+clicking then using the "Exclude" or "Include" buttons; manually selecting records when the current selection is large becomes very inefficient.

Updated 02/12/19

Query By Formula

Demonstrates using a formuala as part of the criteria for selecting records. The formula needs to be an expression that evaluates to either true or false for each record tested. Examples are comparing 2 fields with in a record or using a function provided by the development department.
